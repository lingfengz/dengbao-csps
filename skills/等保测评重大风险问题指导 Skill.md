# 等保测评重大风险问题指导

## 概述
本 **Skill** 用于网络安全等级保护测评过程中的**重大风险隐患判定**。依据《重大风险隐患触发项参照表》（附录G 表-1），结合**相关性、严重性、高发性**三项原则，自动/人工识别测评结果中应判定为重大风险的安全问题，并提供处置建议。

> **适用对象**：等保测评师、安全运维人员、安全管理人员  
> **版本**：V1.0（依据提供表格内容）  
> **依赖**：等保测评报告中的安全问题列表、现场核查记录、渗透测试结果等。

---

## 一、重大风险判定三原则（必同时满足）

| 原则 | 说明 |
|------|------|
| **相关性** | 该安全问题在测评中被发现，且**未采取任何缓解措施**（如补偿控制、临时封堵等）。 |
| **严重性** | 一旦被威胁利用，将导致**业务中断、重要数据泄露/篡改、系统管理权限/业务权限被获取**等严重后果。 |
| **高发性** | 在实际运行环境中，该问题被利用的**概率较大**（如存在公开漏洞、弱口令、暴露面广等）。 |

> **判定流程**：  
> 1. 梳理测评发现的所有安全问题；  
> 2. 逐一对照下方 **触发项参照表**，若匹配则初步标记为候选；  
> 3. 再验证是否**同时满足**上述三原则，若满足则最终确认为**重大风险隐患**。

## 三、使用指导（如何运用本Skill）

### 3.1 输入准备
- **测评报告**中的安全问题清单（含问题描述、位置、影响范围）。
- **现场核查记录**（门禁、机房、设备配置等）。
- **渗透测试/漏洞扫描报告**（高危漏洞列表、弱口令、开放端口等）。
- **运维管理记录**（备份策略、更新记录、资产台账等）。

### 3.2 操作步骤
1. **逐项对照**：从测评问题中提取关键描述，与上表“安全问题描述”进行匹配。  
2. **编码标记**：若匹配成功，标记对应编码（如 `A-18`）。  
3. **三原则验证**：  
   - **相关性**：确认该问题是否未被缓解（例如虽安装杀毒软件但未激活、虽配置门禁但长期失效）。  
   - **严重性**：评估一旦利用是否会导致业务中断、数据泄露、权限丢失等。  
   - **高发性**：判断在当前网络环境（互联网暴露、内部威胁）中，该问题是否易被攻击者利用。  
4. **最终判定**：同时满足三条原则，则将该问题列为 **重大风险隐患**，并写入测评报告风险分析章节。  

### 3.3 输出内容
对于每个判定为重大风险的问题，应输出：
- 问题名称（触发项名称）
- 编码（A-XX）
- 对应测评项（引用）
- 具体安全问题描述（按表内描述）
- 判定依据（三原则符合性说明）
- 建议的整改措施（可选，本表未提供，需根据实际经验补充）

## 四、常见疑难解析

| 场景 | 处理建议 |
|------|---------|
| **同一问题匹配多个触发项** | 取最贴合的一个或多个，均判定为重大风险，但避免重复计算。 |
| **测评项不完全一致但实质相同** | 若实质风险相同（如“未做备份”虽未明确列在表内，但归类为A-26“重要数据无本地备份措施”），应归类到相应项。 |
| **存在缓解措施但效果不足** | 若缓解措施无法完全消除风险（例如有门禁但长期故障），仍可能判定为重大风险，需在“相关性”原则中明确说明。 |
| **综合类触发项（A-27~A-32）** | 这些项通常来源于渗透测试、攻防演练或管理审查，不依赖具体等级保护测评项，可直接依据发现事实触发。 |

## 五、示例

**示例1**：测评发现核心业务系统存在可远程利用的未授权访问漏洞（CVE-2024-XXXX），且未做补丁更新。  
- 匹配编码：**A-22**（存在可被利用的高危安全漏洞）  
- 三原则验证：  
  - 相关：未修补 → 满足；  
  - 严重：可导致数据泄露或系统控制 → 满足；  
  - 高发：漏洞公开，存在攻击工具 → 满足。  
- 结论：**重大风险隐患**。

**示例2**：机房大门使用普通挂锁，无电子门禁。  
- 匹配编码：**A-01**（机房出入口访问控制措施缺失）  
- 三原则验证：  
  - 相关：无任何门禁 → 满足；  
  - 严重：可导致设备被物理破坏或数据窃取 → 满足；  
  - 高发：内部人员或访客易进入 → 满足。  
- 结论：**重大风险隐患**。

## 六、注意事项

- 本表基于**等保2.0**标准及通用风险场景制定，若涉及特殊行业（如金融、政务、能源）还需结合行业额外标准。  
- 部分触发项（如A-03网络设备性能不足）需结合业务峰值流量进行实测验证。  
- **数据备份**类（A-26）特指“本地备份”，若已做异地备份但无本地备份，仍应标记。  
- 对于**综合类**（A-27~A-32），建议通过渗透测试、红蓝演练等主动手段发现，并独立评估。

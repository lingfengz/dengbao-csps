### 一、身份鉴别类（8项）

#### 核查项 LIN-AUTH-001：密码复杂度策略

```markdown
| 属性 | 内容 |
|-----|------|
| **核查编号** | LIN-AUTH-001 |
| **核查项** | 密码复杂度策略是否配置 |
| **等保条款** | 安全计算环境-身份鉴别（应对登录的用户进行身份标识和鉴别，鉴别信息应具有复杂度要求并定期更换） |
| **风险等级** | 🔴 高风险（如不符合） |

### 核查方法

**手动检查**：
```bash
# 检查 pam_cracklib 或 pam_pwquality 配置
cat /etc/pam.d/system-auth | grep -E "pam_cracklib|pam_pwquality"

# 检查密码复杂度参数
cat /etc/security/pwquality.conf | grep -v "^#" | grep -v "^$"
```

**自动化检查脚本**：
```bash
#!/bin/bash
# 检查密码复杂度配置

echo "=== 密码复杂度策略检查 ==="

# 检查pam模块
if grep -q "pam_pwquality.so" /etc/pam.d/system-auth 2>/dev/null; then
    echo "[INFO] pam_pwquality 已配置"
    # 检查参数
    grep "pam_pwquality.so" /etc/pam.d/system-auth
elif grep -q "pam_cracklib.so" /etc/pam.d/system-auth 2>/dev/null; then
    echo "[INFO] pam_cracklib 已配置"
    grep "pam_cracklib.so" /etc/pam.d/system-auth
else
    echo "[FAIL] 未配置密码复杂度模块"
fi

# 检查pwquality配置
if [ -f /etc/security/pwquality.conf ]; then
    echo "=== pwquality.conf 配置 ==="
    grep -E "minlen|dcredit|ucredit|lcredit|ocredit" /etc/security/pwquality.conf
fi
```

### 判定标准

| 检查点 | 三级要求 | 二级要求 |
|-------|---------|---------|
| 密码最小长度 | ≥8位 | ≥8位 |
| 包含大写字母 | 是（dcredit=-1） | 建议 |
| 包含小写字母 | 是（lcredit=-1） | 建议 |
| 包含数字 | 是（ucredit=-1） | 建议 |
| 包含特殊字符 | 是（ocredit=-1） | 建议 |
| 至少包含3类字符 | 是 | 是 |

### 加固方案

```bash
# 方式一：使用 authconfig（CentOS 7）
authconfig --passminlen=8 --passmaxrepeat=3 --enablereqlower \
           --enablerequpper --enablereqdigit --enablereqother --update

# 方式二：编辑 /etc/security/pwquality.conf
cat >> /etc/security/pwquality.conf << EOF
minlen = 8
dcredit = -1
ucredit = -1
lcredit = -1
ocredit = -1
minclass = 3
maxrepeat = 3
EOF

# 方式三：编辑 /etc/pam.d/system-auth
# 在 password 行添加 pam_pwquality.so 参数
# password requisite pam_pwquality.so try_first_pass local_users_only retry=3 authtok_type=
```

```markdown
| 属性 | 内容 |

| **核查项** | 密码有效期是否配置 |
| **等保条款** | 安全计算环境-身份鉴别（鉴别信息应定期更换） |

```bash
# 设置密码有效期
sed -i 's/^PASS_MAX_DAYS.*/PASS_MAX_DAYS 90/' /etc/login.defs

sed -i 's/^PASS_WARN_AGE.*/PASS_WARN_AGE 7/' /etc/login.defs

# 对现有用户应用策略

#### 核查项 LIN-AUTH-003：登录失败锁定策略

```markdown
| 属性 | 内容 |
|-----|------|
| **核查编号** | LIN-AUTH-003 |
| **核查项** | 登录失败锁定策略 |
| **等保条款** | 安全计算环境-身份鉴别（应启用登录失败处理功能） |
| **风险等级** | 🔴 高风险（如不符合） |

### 核查方法

```bash
# 检查 pam_tally2 或 pam_faillock 配置
cat /etc/pam.d/system-auth | grep -E "pam_tally2|pam_faillock"
cat /etc/pam.d/password-auth | grep -E "pam_tally2|pam_faillock"

# CentOS 8/RHEL 8 使用 pam_faillock
cat /etc/security/faillock.conf 2>/dev/null
```

### 判定标准

| 参数 | 三级要求 | 二级要求 |
|-----|---------|---------|
| 失败锁定次数 | ≤5次 | ≤10次 |
| 锁定时间 | ≥15分钟 | ≥5分钟 |
| 解锁方式 | 手动或定时 | 定时 |

### 加固方案（CentOS 7）

```bash
# 在 /etc/pam.d/system-auth 的 auth 段落添加
# CentOS 7 使用 pam_tally2
cat >> /etc/pam.d/system-auth << 'EOF'
auth required pam_tally2.so deny=5 unlock_time=900 even_deny_root root_unlock_time=900
EOF

# 在 account 段落添加
cat >> /etc/pam.d/system-auth << 'EOF'
account required pam_tally2.so
EOF
```

### 加固方案（CentOS 8/RHEL 8+）

```bash
# 编辑 /etc/security/faillock.conf
cat >> /etc/security/faillock.conf << EOF
deny = 5
unlock_time = 900
even_deny_root
root_unlock_time = 900
EOF

# 在 /etc/pam.d/system-auth 添加
auth required pam_faillock.so preauth
auth required pam_faillock.so authfail
account required pam_faillock.so
```

```markdown
| 编号 | 核查项 | 关键命令 | 三级要求 | 
|-----|-------|---------|---------|
| LIN-AUTH-004 | root远程登录限制 | `grep PermitRootLogin /etc/ssh/sshd_config` | 禁止root SSH登录 |
| LIN-AUTH-005 | 空密码账户检查 | `awk -F: '($2==""){print $1}' /etc/shadow` | 无空密码账户 |

| LIN-AUTH-008 | 会话超时配置 | `grep TMOUT /etc/profile` | TMOUT≤600秒 |

### 加固脚本（身份鉴别一键加固）

```bash
#!/bin/bash
# Linux 身份鉴别一键加固脚本
# 适用：CentOS 7/8, RHEL 7/8
# 等级：等保三级

set -e

echo "========================================="
echo "  Linux 身份鉴别安全加固开始"
echo "  $(date)"
echo "========================================="

# 1. 密码复杂度
echo "[1/8] 配置密码复杂度..."
cat > /etc/security/pwquality.conf << EOF
minlen = 8
dcredit = -1
ucredit = -1
lcredit = -1
ocredit = -1
minclass = 3
maxrepeat = 3
EOF

# 2. 密码有效期
echo "[2/8] 配置密码有效期..."
sed -i 's/^PASS_MAX_DAYS.*/PASS_MAX_DAYS   90/' /etc/login.defs
sed -i 's/^PASS_MIN_DAYS.*/PASS_MIN_DAYS   1/' /etc/login.defs
sed -i 's/^PASS_WARN_AGE.*/PASS_WARN_AGE   7/' /etc/login.defs

# 3. 登录失败锁定
echo "[3/8] 配置登录失败锁定..."
cat >> /etc/pam.d/system-auth << 'EOF'
auth required pam_tally2.so deny=5 unlock_time=900 even_deny_root root_unlock_time=900
account required pam_tally2.so
EOF

# 4. 禁止root SSH登录
echo "[4/8] 禁止root SSH登录..."
sed -i 's/^#PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sed -i 's/^PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config

# 5. 检查空密码账户
echo "[5/8] 检查空密码账户..."
EMPTY_USERS=$(awk -F: '($2==""){print $1}' /etc/shadow)
if [ -n "$EMPTY_USERS" ]; then
    echo "[WARNING] 发现空密码账户: $EMPTY_USERS"
    echo "请手动处理这些账户"
fi

# 6. 检查UID唯一性
echo "[6/8] 检查UID唯一性..."
DUP_UID=$(awk -F: '{print $3}' /etc/passwd | sort | uniq -d)
if [ -n "$DUP_UID" ]; then
    echo "[WARNING] 发现重复UID: $DUP_UID"
fi

# 7. SSH协议版本
echo "[7/8] 加固SSH配置..."
echo "Protocol 2" >> /etc/ssh/sshd_config
# 其他SSH加固
sed -i 's/^#MaxAuthTries.*/MaxAuthTries 5/' /etc/ssh/sshd_config
sed -i 's/^MaxAuthTries.*/MaxAuthTries 5/' /etc/ssh/sshd_config
sed -i 's/^#ClientAliveInterval.*/ClientAliveInterval 300/' /etc/ssh/sshd_config
sed -i 's/^#ClientAliveCountMax.*/ClientAliveCountMax 0/' /etc/ssh/sshd_config

# 8. 会话超时
echo "[8/8] 配置会话超时..."
echo "export TMOUT=600" >> /etc/profile
echo "readonly TMOUT" >> /etc/profile

# 重启SSH服务
systemctl restart sshd 2>/dev/null || service sshd restart 2>/dev/null

echo "========================================="
echo "  身份鉴别安全加固完成"
echo "  请在测试后重启SSH服务"
echo "========================================="
```

### 审计加固脚本

```bash
#!/bin/bash
echo "=== 安全审计加固 ==="

# 1. 安装并启动auditd
yum install audit -y
systemctl enable auditd
systemctl start auditd

# 2. 配置审计规则
cat > /etc/audit/rules.d/security.rules << 'EOF'
# 记录用户登录事件
-w /var/log/lastlog -p wa -k logins
-w /var/run/faillock -p wa -k logins

# 记录账号修改
-w /etc/passwd -p wa -k identity
-w /etc/shadow -p wa -k identity
-w /etc/group -p wa -k identity

# 记录sudo使用
-w /etc/sudoers -p wa -k sudoers
-w /var/log/sudo.log -p wa -k sudo

# 记录时间修改
-a always,exit -F arch=b64 -S adjtimex -S settimeofday -k time_change
-a always,exit -F arch=b32 -S adjtimex -S settimeofday -k time_change

# 记录特权命令
-a always,exit -F arch=b64 -F euid=0 -S execve -k rootcmd
-a always,exit -F arch=b32 -F euid=0 -S execve -k rootcmd

# 设置审计配置不可变（最后一行）
-e 2
EOF

# 重启auditd
service auditd restart

# 3. 配置日志远程转发
# 替换为实际日志服务器IP
echo "*.* @192.168.1.100:514" >> /etc/rsyslog.conf
systemctl restart rsyslog

# 4. 配置时间同步
yum install chrony -y
systemctl enable chronyd
systemctl start chronyd
echo "server ntp.example.com iburst" >> /etc/chrony.conf

echo "安全审计加固完成"

### 入侵防范加固脚本

```bash
#!/bin/bash
echo "=== 入侵防范加固 ==="

# 1. 启用防火墙
systemctl enable firewalld
systemctl start firewalld
firewall-cmd --set-default-zone=drop
firewall-cmd --permanent --add-service=ssh
firewall-cmd --reload

# 2. 内核参数加固
cat >> /etc/sysctl.conf << 'EOF'
# IP Spoofing protection
net.ipv4.conf.all.rp_filter = 1
net.ipv4.conf.default.rp_filter = 1

#!/bin/bash
# MySQL 安全配置核查脚本
# 使用方法：bash mysql_check.sh <host> <port> <user> <password>

# Nginx 等保三级安全配置
server {
    listen 443 ssl http2;
    
    # 1. 隐藏版本号
    server_tokens off;
    
    # 2. 限制HTTP方法
    if ($request_method !~ ^(GET|HEAD|POST)$) {
        return 405;
    }
    
    # 3. 安全响应头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self';" always;

    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;
    
    # 5. 目录列表禁止
    autoindex off;
    
    # 6. 缓冲区限制
    client_body_buffer_size 128k;
    client_header_buffer_size 4k;
    client_max_body_size 10m;
    large_client_header_buffers 4 8k;
    
    # 7. 超时设置
    client_body_timeout 30s;
    client_header_timeout 30s;
    keepalive_timeout 30s;
    send_timeout 30s;
    
    # 8. 日志配置
    access_log /var/log/nginx/access.log combined;
    error_log /var/log/nginx/error.log warn;
    
    location / {
        # 9. 安全配置
        try_files $uri $uri/ =404;
    }
    
    # 10. 禁止访问隐藏文件
    location ~ /\. {
        deny all;
        access_log off;
        log_not_found off;
    }
}
